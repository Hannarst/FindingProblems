Add a problem

Problems and their solutions can be added to the database using one of two methods:

  • Manually

    • Fill out and submit this form. You may use markdown in the content and the solution fields. The execution time limit is the maximum length of time a solution can run to be considered a successful solution to this problem. This is done to exclude brute force solutions if necessary. The problems you add are private by default and can only be seen by logged in users (CS staff), but uncheck this box if you want students to be able to practice this question on their own time. Categorise this problem as you see fit.

  • Uploading PDFs

    Problems and solutions can also be created by uploading PDF documents. Each document may contain only one problem or solution. This is enforced to allow the user time to check that the system has created a problem or solution correctly from the uploaded PDF.

Editing vs forking a problem

Editing () a problem rewrites the problem, while forking () a problem allows you to create a new problem from the modification. The original stays the same. You can later see which problem a particular problem has been forked off in the Forked From field.

Markdown

Problems and solutions can be styled using Markdown. All Markdown styles are supported, courtesy of Python's markdown2. A full description of Markdown and its syntax is provided by its inventor here.
